# Deploy Recipe - GitHub Pages

This recipe applies to projects generated by Auralith Stack templates.

## 1) Create repository

- Push your generated app to GitHub.
- Keep default branch as `main`.

## 2) Configure repository secrets

Go to:

- `Settings -> Secrets and variables -> Actions`

Add:

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_PUBLISHABLE_KEY`
- `VITE_SUPABASE_CONTACT_FUNCTION`
- `VITE_CONTACT_EMAIL`
- `VITE_FORMSUBMIT_TOKEN`

## 3) Configure Pages source

Go to:

- `Settings -> Pages`

Set:

- Source: `Deploy from a branch`
- Branch: `gh-pages`
- Folder: `/ (root)`

## 4) Trigger deploy

Push to `main`:

```bash
git add .
git commit -m "chore: deploy setup"
git push
```

The workflow will:

1. install dependencies,
2. set `VITE_BASE_PATH` automatically,
3. build,
4. publish `dist` to `gh-pages`.

## 5) Verify production

- Open the published URL.
- Check static assets and routing.
- Submit a contact form test:
  - confirm lead in Supabase
  - confirm fallback email path (if configured)
