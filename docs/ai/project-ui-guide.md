# Generated Project UI Guide

This guide should be used inside projects generated by Auralith Stack.

## Source of Truth

Use `auralith-ui` for reusable UI components.

Use Tailwind classes for layout and page composition.

Use project-specific files only for domain logic, data, pages, and feature composition.

## Do Not Recreate

Do not recreate these primitives in generated projects:

- Button
- Card
- Input
- Textarea
- Select
- Modal
- AlertDialog
- DropdownMenu
- Tooltip
- Popover
- Toast
- Navbar
- SideRail
- DataTable

## Styling Policy

Prefer inline Tailwind class composition:

```tsx
<Card className="grid gap-4 p-4 md:grid-cols-2">
  ...
</Card>
```

Avoid local CSS files for component styling.

Keep global CSS limited to imports, tokens, resets, and browser-level behavior.

## Component Decisions

- Use `Navbar` for landing navigation.
- Use `SideRail` for dashboard navigation.
- Use `Card` for grouped content.
- Use `GlassPanel` for hero or premium surfaces.
- Use `Input` and `Textarea` for forms.
- Use `SearchInput` for search/filter fields.
- Use `DropdownMenu` for contextual actions.
- Use `Tooltip` only for short non-interactive hints.
- Use `Popover` for interactive floating content.
- Use `Modal` for short blocking workflows.
- Use `AlertDialog` for destructive confirmation.
- Use `Toast` for non-blocking feedback.

## Theme

Use the Auralith theme tokens and `data-theme` on the root element.

Prefer tokens such as:

- `--text-main`
- `--text-soft`
- `--text-muted`
- `--surface-panel-3`
- `--surface-menu`
- `--card-border`
- `--input-bg`
- `--accent-line`

## AI Agent Checklist

Before adding new UI:

1. Check if Auralith UI already has the component.
2. Use Tailwind classes for layout.
3. Avoid a new CSS file.
4. Use the correct overlay component.
5. Run the project build.
